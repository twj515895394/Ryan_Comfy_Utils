# Ryan Batch Video Loader：添加视频预览功能（显示/隐藏/播放控制）

## 父问题
`.scratch/comfy-node-ux/PRD.md`

## 要构建什么
为 `Ryan Batch Video Loader` 节点添加类似官方/社区视频加载节点的预览能力：
- 节点上直接显示选中的视频（缩略或播放器）
- 提供 UI 控件：隐藏/显示视频预览
- 提供播放/暂停控制（或利用浏览器原生 video 控件）
- 切换 index 或文件时，预览更新为当前选中视频
- 预览不影响节点输出（仍输出 IMAGE 帧批次 + 元数据）

实现方式：前端 JS 使用 ComfyUI 的 addDOMWidget 创建 <video> 元素，监听 index / 选中文件变化，设置 video.src 为本地文件路径（需处理 file:// 或通过 ComfyUI 静态服务暴露）。

注意：ComfyUI 自定义节点通常无法直接用 file:// 播放本地视频，需研究如何将视频路径映射到可被前端访问的 URL（可能通过 /view 或自定义静态路由，或先输出缩略图+播放控制占位）。

端到端：用户在节点上看到当前视频预览，可隐藏/显示、播放暂停；切换视频后预览刷新；不破坏现有输出。

## 验收标准
- [ ] 节点上出现视频预览区域（DOM widget）
- [ ] 支持隐藏/显示预览
- [ ] 支持播放/暂停（或原生控件）
- [ ] 切换 index / 文件时预览更新
- [ ] 节点功能（输出 frames + metadata）不受影响
- [ ] 参考 comfyui-easy-use 或类似社区节点的实现模式

## 被阻塞于
无 - 可以立即开始（但需调研 ComfyUI 前端视频资源暴露机制）

## 评论
- 技术难点：本地视频文件如何被浏览器 <video> 标签直接访问。可能需要：
  - 将视频复制/链接到 output 目录并用相对 URL
  - 或实现简单后端路由暴露输入目录下的视频
  - 或退化为首帧缩略图 + 外部播放链接
- 建议先实现基础 widget 框架，路径问题可迭代
