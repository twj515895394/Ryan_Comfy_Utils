# Handoff: 视频分镜与 ACP 节点稳定化工作交接

## 1. 任务当前状态与进展

我们最近在 `Ryan_Comfy_Utils` 自定义节点包中完成了以下工作：

- **视频分镜物理切割系统 (`Ryan Video Scene Splitter`)**：
  - 基于 PySceneDetect 完成了分镜检测、碎片镜头合并、首帧提取的核心功能开发（[scene_detect.py](file:///x:/ComfyUI-aki-v2/ComfyUI/custom_nodes/Ryan_Comfy_Utils/ryan_comfy_utils/core/scene_detect.py)）。
  - 在节点输入中完整添加了与官方/主流加载节点完全一致的 6 个高阶控制参数：`force_rate`, `custom_width`, `custom_height`, `frame_load_cap`, `skip_first_frames`, `select_every_nth`（[video_nodes.py](file:///x:/ComfyUI-aki-v2/ComfyUI/custom_nodes/Ryan_Comfy_Utils/ryan_comfy_utils/nodes/video_nodes.py)）。
  - 支持在重编码模式下（非 Fast Copy）调用 FFmpeg 进行强制帧率转换、分辨率缩放和抽帧采样过滤。
  - 前端 JavaScript（[ryan_scene_splitter.js](file:///x:/ComfyUI-aki-v2/ComfyUI/custom_nodes/Ryan_Comfy_Utils/ryan_comfy_utils/web/ryan_scene_splitter.js)）实现了动态从后端加载视频属性，友好显示并在控制项标签上标注实际视频参数的能力（原视频尺寸/总帧数/帧率）。

- **多端兼容与健壮性稳定化（核心修复）**：
  - **Windows CLI 执行挂起/文件找不到错误**：修复了 `Ryan ACP Universal Agent` 调用 `subprocess` 运行 Claude CLI 无法解析 npm 包装脚本（`.cmd` / `.bat`）的 [WinError 2] 崩溃问题（通过在 Windows 平台智能启用 `shell=True` 并通过了 87 项全量单元测试）。
  - **ComfyUI 历史工作流 `NaN` / 校验失败**：修复了加载老版本工作流时，由于 widget 数量改变导致索引错位，将视频 URL 解析成 `custom_width` 引发的 ComfyUI 底层 `validate_prompt` 类型强转校验失败。前端 `configure` 方法新增了补位和自动类型修复，后端 `run` 增强了对 `None` / 异常类型的防御性降级。

- **文档补全**：
  - 已经完整补齐了 [README.md](file:///x:/ComfyUI-aki-v2/ComfyUI/custom_nodes/Ryan_Comfy_Utils/README.md) 关于视频分镜切割节点 (`Ryan Video Scene Splitter`) 和图像批次拆分节点 (`Ryan Image Batch Splitter`) 的详细输入输出参数说明。
